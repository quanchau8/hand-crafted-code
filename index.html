Our Conviction

On writing every character

A line of code deserves the full attention of the person writing it. That is the whole philosophy of the house. Everything else here follows from it.

Somewhere along the way, the industry began to accept suggestions. Grey text appeared at the end of the cursor, and programmers learned to press Tab. We tried it once, in 2021, on a machine kept outside the building for the purpose. The suggestions were fine. This is what worried us.

Our artisans write in silence, from memory. The editor is plain. The cursor sits where it was left and offers no opinion. If a function takes all afternoon, it takes all afternoon. The afternoon was going to pass regardless.

The motto of the house is festina lente. Make haste slowly. It has not been revised since 2009.

The Method

Our process

  1. I

    Sourcing

    We recruit developers with no history of assisted completion. Each candidate writes a linked list in pencil while a senior artisan watches. Fewer than one in two hundred is invited to the bench.

  2. II

    Resting

    Freshly written code rests overnight in a cool, dark branch. This allows the logic to settle and the off-by-ones to rise gently to the surface, where they belong.

  3. III

    Signing

    Every git blame is signed by hand. The author's name appears in the history because a person put it there on purpose.

  4. IV

    Review by candlelight

    Batches of no more than forty lines are read aloud by candlelight until the room agrees. Some batches are never agreed upon. Those are composted.

On pasting

The clipboard was removed from the machines in 2013. When an artisan needs an answer from Stack Overflow, she transcribes it by hand, character by character. Errors introduced in the copying belong to the house. Several functions can be dated by them.

Provenance

Heritage features

Every batch leaves the atelier with small irregularities. Buyers should learn to recognize them, since counterfeit hand-written code has become common.

Trace technical debt

Contains trace amounts of technical debt, as nature intended. Bottled unfined and unfiltered. Sediment in the older files is normal and should not be disturbed.

The load-bearing global

Somewhere in every batch lives a single global variable named after the artisan's cat. It holds the entire structure up. Renaming it voids the warranty and, in two documented cases, the build.

The untouched function

One function arrives marked // do not touch. It has passed intact through four generations of maintainers. It works. Nobody now living knows why.

Terroir naming

Variable names express the conditions of their vintage: temp, tempFINAL, actualTemp_USE_THIS_ONE. The 2019 pressing reads poorly and we no longer discuss it.

Documentation is passed down orally, at meals.
